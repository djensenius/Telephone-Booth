# ADR 0005 — Rust 1.95.0 toolchain

**Status:** accepted.

## Context

We need a single pinned Rust version for reproducibility (CI, the `.deb`,
locally-developed branches). 1.95.0 is requested by the project owner.

## Decision

- Pin Rust **1.95.0** in `rust-toolchain.toml`.
- `mise.toml` mirrors that pin so the rest of the toolchain (cargo-deny,
  cargo-deb, cross, etc.) is bootstrapped consistently.
- CI uses the same pin via `dtolnay/rust-toolchain@master` with
  `toolchain: 1.95.0`.

## Consequences

**Good:**

- Reproducible builds across laptop, CI, and Pi.
- Lets us use language features through edition 2024 + 1.95 stable
  without per-crate gating.

**Trade-offs:**

- Need to bump deliberately. The pin change is one line + a CI rerun.
- Some crates may require feature flags to compile on 1.95; the
  workspace lock pins compatible versions.
